## 1. 원격 저장소 복제 (`$ git clone`)

### 1.1 `clone` 개념 및 명령
* [cite_start]**정의**: 깃허브(원격 저장소)에 있는 코드를 **지역 저장소**에 **복제(clone)**하는 명령입니다. [cite: 6354]
* [cite_start]**특징**: 공개된 저장소는 소유와 관계없이 누구나 복제 가능합니다. [cite: 6355, 6344]
* [cite_start]**원격 저장소 별칭 (Alias)**: `clone`을 수행하면 자동으로 원격 저장소 주소에 **`origin`**이라는 별칭이 설정됩니다. [cite: 6391]

| 명령어 | 설명 |
| :--- | :--- |
| **`$ git clone <주소>`** (예: `.../repo.git`) | [cite_start]원격 저장소와 동일한 이름의 폴더를 생성하고 복제합니다. [cite: 6371] |
| **`$ git clone <주소> <폴더명>`** | [cite_start]지정한 이름의 하부 폴더에 복제합니다. [cite: 6373] |
| **`$ git clone <주소> .`** | [cite_start]현재 폴더에 바로 복제합니다. [cite: 6375] |

### 1.2 원격 저장소 관리 명령
* [cite_start]**`$ git remote`**: 현재 지역 저장소에 설정된 **원격 저장소의 별칭 목록**만 표시합니다. [cite: 6389, 6552]
* [cite_start]**`$ git remote -v`**: 원격 저장소의 **별칭 및 주소** 목록을 상세히 표시합니다. [cite: 6393, 6541]
* [cite_start]**`$ git remote show <별칭>`**: 지정한 원격 저장소의 자세한 정보(Fetch/Push URL, HEAD 브랜치, 추적 중인 브랜치 등)를 표시합니다. [cite: 6562, 6586, 6614]
* [cite_start]**`$ git remote rename <기존별칭> <새별칭>`**: 원격 저장소의 별칭 이름을 수정합니다. [cite: 6566]
* [cite_start]**`$ git remote rm <별칭>`**: 원격 저장소 별칭을 삭제합니다. [cite: 6568]

## 2. 개인 접근 토큰 (PAT) 인증

### 2.1 PAT 개요
* [cite_start]**필요성**: 2021년 8월 13일 이후부터 깃허브에서 기존의 **비밀번호 인증 대신** **개인 접근 토큰(PAT, Personal Access Token)**을 사용하도록 변경되었습니다. [cite: 37, 40, 42]
* [cite_start]**발급**: 깃허브 계정마다 토큰을 생성해야 하며, 생성 후에는 **다시 표시되지 않으므로** 잘 보관해야 합니다. [cite: 44, 325, 346]
* [cite_start]**범위(Scopes)**: 토큰 생성 시 접근 권한 범위를 설정해야 하며, 일반적으로 `repo`, `workflow`, `gist` 정도를 선택합니다. [cite: 230, 281, 285]
* **사용 방법**: 인증 오류 시 PAT를 사용하여 `push`를 시도할 수 있습니다.
    * [cite_start]`$ git push -u https://{token}@{username}/{repo_name}.git` [cite: 461, 471]

## 3. 지역-원격 저장소 동기화 (Push, Pull, Fetch)

### 3.1 Push (지역 → 원격)
* [cite_start]**정의**: 로컬 저장소에 남겨놓은 코드 변경 이력들을 **원격 저장소로 전송**합니다 (올리기, 밀기). [cite: 501]
* [cite_start]**권한**: 자신의 저장소이거나 다른 사람의 저장소일 경우 협업자(collaborator)로 등록되어 **쓰기 권한**이 있어야 `push`가 가능합니다. [cite: 498, 499]
* **명령**:
    * [cite_start]`$ git push <저장소별칭명> <브랜치명>` (예: `$ git push origin main`) [cite: 503]
    * [cite_start]**`$ git push`**: 최초 `push` 시 `-u` 옵션을 사용하여 저장소명과 브랜치명을 지정한 후에는 인자 생략이 가능합니다. [cite: 524, 527]
    * [cite_start]**설정**: `$ git config --global push.default current`로 설정하면 어느 브랜치에서 작업하든 `$ git push`만으로 원격 저장소의 동일한 브랜치로 업로드가 가능합니다. [cite: 531, 533]

### 3.2 Pull vs. Fetch (원격 → 지역)
| 명령어 | 역할 | 동작 방식 |
| :--- | :--- | :--- |
| **`$ git pull`** | [cite_start]원격 저장소의 수정을 지역 저장소에 **반영**합니다 (내리기, 끌기). [cite: 391, 563] | [cite_start]**`$ git fetch` 명령과 `$ git merge` 명령이 순차적으로 진행**됩니다. [cite: 544, 545] |
| **`$ git fetch`** | [cite_start]원격 저장소의 정보를 **로컬(remote/origin) 저장소로만 가져옵니다.** [cite: 547, 572] | **병합(Merge)을 미수행**합니다. [cite_start]변경된 내용을 확인한 후 병합 여부를 결정할 때 유용합니다. [cite: 550, 572, 573] |
| **`$ git merge origin/<branch>`** | [cite_start]`fetch`로 가져온 원격 저장소의 정보를 **지역 저장소의 내용과 병합**합니다. [cite: 549] | [cite_start]`fetch` 이후 지역 저장소에 원격의 내용을 반영할 때 사용합니다. [cite: 600, 601]

## 4. 기타 원격 연동 (Clone with VS Code)

* [cite_start]**VS Code에서 복제**: VS Code 시작 화면이나 명령 팔레트에서 **`Clone Git Repository`**를 선택한 후, 깃 주소를 입력하여 원격 저장소를 복제할 수 있습니다. [cite: 6761, 6782, 6762]
