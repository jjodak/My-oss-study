## 1. 깃 설치 및 실행

### 1.1 깃 다운로드 및 설치
* [cite_start]**다운로드 사이트**: 깃 공식 홈페이지(https://git-scm.com/downloads)에서 내려받을 수 있습니다[cite: 2513].
* [cite_start]**설치 파일**: 일반적으로 `Git-#.##.#-64-bit.exe` 형태의 설치 파일을 사용합니다[cite: 1963, 2516].
* [cite_start]**설치 경로**: 설치 성공 시 기본 경로는 `C:\Program Files\Git`입니다[cite: 2042, 2157, 2519].
* [cite_start]**기본 브랜치 설정**: 새 저장소의 기본 브랜치 이름은 "master"에서 **"main"**으로 설정하는 것이 권장됩니다[cite: 2120, 2124].

### 1.2 깃 실행 방법
[cite_start]깃 설치 후 다음 두 가지 소프트웨어를 제공합니다: **Git Bash**와 **Git GUI**[cite: 1686].

* [cite_start]**시작 메뉴에서 실행**: 시작 메뉴에서 `Git Bash`를 검색하여 실행할 수 있습니다[cite: 2294].
* [cite_start]**탐색기 폴더에서 실행**: 원하는 폴더에서 마우스 오른쪽 버튼을 클릭하여 팝업 메뉴 중 **'Git Bash Here'**를 선택하면 해당 폴더에서 바로 깃 배시가 실행됩니다[cite: 2324, 2345].

### 1.3 깃 명령어 기초
* [cite_start]**버전 확인**: `git --version` 또는 `git version`[cite: 2495, 2525].
* [cite_start]**설정 확인**: `git config --list`[cite: 2497].
* [cite_start]**도움말 보기**: `git --help`[cite: 2499].

## 2. 깃 설정 및 저장소 생성

### 2.1 깃 설정 명령 구조 및 범위
* [cite_start]**명령 구조**: `$ git config --설정범위 설정변수 설정값`[cite: 507, 711].
* [cite_start]**설정 범위**: [cite: 509, 713]
    * [cite_start]`--system`: **모든 사용자**에게 적용됩니다[cite: 511, 514].
    * [cite_start]`--global`: **현재 사용자**의 **모든 저장소**에 적용됩니다[cite: 512, 515].
        * [cite_start]설정 내용은 `C:\Users\[사용자 계정]\.gitconfig` 파일에 저장됩니다[cite: 534, 715].
    * [cite_start]`--local`: 현재 사용자의 **현재 저장소**에만 적용됩니다[cite: 513, 516].

### 2.2 저장소 생성 전 필수 설정 (Global)
[cite_start]저장소 생성 전에 다음 **6가지 설정**을 확인하고 실행하는 것이 권장됩니다[cite: 587, 592, 720].

| 설정변수 | 명령 예시 | 설명 |
| :--- | :--- | :--- |
| `user.name` | `$ git config --global user.name "ai7dnn"` | [cite_start]**사용자 이름** 설정[cite: 492, 592]. |
| `user.email` | `$ git config --global user.email ai7dnn@gmail.com` | [cite_start]**사용자 전자메일** 설정[cite: 492, 592]. |
| `init.defaultBranch` | `$ git config --global init.defaultBranch main` | [cite_start]**기본 브랜치 이름** 설정[cite: 523, 592, 731]. |
| `core.editor` | `$ git config --global core.editor 'code --wait'` | [cite_start]**기본 편집기** 설정 (예: VS Code)[cite: 592, 729]. |
| `core.autocrlf` | `$ git config --global core.autocrlf true` | [cite_start]**줄바꿈 자동변환** (Windows에서 true 권장)[cite: 557, 565, 592, 725]. |
| `core.safecrlf` | `$ git config --global core.safecrlf false` | [cite_start]**줄바꿈 안전 설정** (경고 발생 없애기 옵션)[cite: 572, 580, 592, 727]. |

> [cite_start]💡 **줄바꿈 설정**: Windows는 `CR`과 `LF`를, Mac/Linux는 `LF`를 줄 끝으로 인식하기 때문에 플랫폼 간의 협업을 위해 `core.autocrlf` 설정이 필요합니다[cite: 561, 562, 563].

### 2.3 깃 저장소 생성 (`git init`)
[cite_start]디렉토리를 Git Repository로 만들어야 버전 관리가 가능합니다[cite: 610].

* `$ git init` 또는 `$ git init .`
    * [cite_start]**현재 디렉토리를** Git Repository로 만듭니다[cite: 614, 737, 738].
* `$ git init [폴더명]` (예: `$ git init basic`)
    * [cite_start]현재 폴더 하부에 새로운 폴더를 생성하고 이를 Git Repository로 만듭니다[cite: 616, 739, 740].

> [cite_start]⚠️ **저장소 확인**: 저장소로 생성된 폴더 하부에는 반드시 **`.git` 폴더**가 포함되며 [cite: 639, 647][cite_start], 이 폴더를 삭제하면 깃 저장소로서의 기능을 상실합니다[cite: 651]. [cite_start]커밋된 모든 파일의 버전 기록이 `.git` 폴더에 저장됩니다[cite: 649].

## 3. 비주얼 스튜디오 코드(VS Code)와 리눅스 명령

### 3.1 VS Code 개요 및 활용
* [cite_start]**개요**: MS사가 개발한 오픈소스 에디터 소프트웨어로, 가볍고 강력한 소스 코드 편집기입니다[cite: 1078, 1080].
* [cite_start]**버전 관리 지원**: 파일 편집과 더불어 **깃 버전 관리 기능도 지원**합니다[cite: 1088, 1662].
* **설치 권장 사항**: 설치 과정 중 탐색기에서 파일 또는 디렉토리의 팝업 메뉴에 **'Code(으)로 열기'** 메뉴를 추가하는 것을 권장합니다. [cite_start]이를 통해 탐색기에서 바로 VS Code를 사용할 수 있어 편리합니다[cite: 1255, 1265, 1266, 1337, 1664].

### 3.2 깃 Bash에서 사용되는 주요 리눅스 명령
[cite_start]깃 CLI 환경인 Git Bash에서는 깃 명령과 함께 리눅스 기초 명령어를 사용합니다[cite: 1667].

| 영역 | 명령 | 설명 |
| :--- | :--- | :--- |
| **폴더** | `pwd` | [cite_start]현재 작업 디렉토리 표시 (Print Working Directory)[cite: 1497, 1499, 1668]. |
| | `ls -al` | [cite_start]파일 및 폴더 목록 상세 정보 표시 (`-a`: 숨김 파일, `-l`: 상세 정보)[cite: 1507, 1508, 1509, 1575, 1577, 1669]. |
| | `cd` | [cite_start]디렉토리 이동 (Change Directory)[cite: 1500, 1501]. |
| | `mkdir [dname]` | [cite_start]디렉토리 생성 (Make Directory)[cite: 1502, 1503]. |
| **파일** | `touch [fname]` | [cite_start]빈 파일 생성[cite: 1523, 1524, 1670]. |
| | `cat [fname]` | [cite_start]파일 내용 화면에 출력 (Concatenate)[cite: 1540, 1541, 1597, 1672]. |
| | `echo [문자열]` | [cite_start]문자열을 컴퓨터 터미널에 출력[cite: 1526, 1528]. |
| | `cp a b` | [cite_start]파일 a를 b로 복사[cite: 1542, 1543]. |
| | `mv f1 f2` | [cite_start]파일 f1을 f2로 이름 수정 또는 이동[cite: 1544, 1545]. |
| **삭제** | `rm [fname]` | [cite_start]파일 삭제[cite: 1557, 1558]. |
| | `rm -rf [dname]` | [cite_start]디렉토리 및 하위 내용 강제 삭제 (`-r`: 내부 전체 삭제, `-f`: 강제 삭제)[cite: 1559, 1561, 1562]. |
| **연산** | `A \| B` | [cite_start]**파이프(Pipe)**: 앞 명령(A)의 표준 출력을 뒤 명령(B)의 표준 입력으로 처리[cite: 1603]. |
| | `echo aaa > a.txt` | [cite_start]**리다이렉션** (`>`): 출력 결과를 파일로 저장(기존 내용 **지우고** 저장)[cite: 1618, 1619, 1632, 1671]. |
| | `echo bbb >> a.txt` | [cite_start]**리다이렉션** (`>>`): 출력 결과를 파일 **뒤에 덧붙여서** 저장[cite: 1620, 1621, 1634]. |
