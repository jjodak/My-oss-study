### 📝 Repository Overview
이 저장소는 **오픈소스 소프트웨어(OSS)** 강좌를 수강하며 학습한 핵심 이론과 실습 내용을 기록하는 공간입니다. 버전 관리 시스템인 Git의 기초부터 GitHub를 활용한 협업 프로세스, 그리고 오픈소스 생태계의 라이선스 체계까지 실무적인 역량을 쌓는 과정을 담고 있습니다.

### 📅 주차별 학습 요약 (Weekly Study)

| 주차 | 주요 학습 내용 | 핵심 키워드 |
| :--- | :--- | :--- |
| **Week 2** | 오픈소스와 Git의 시작 | 환경 설정, `init`, `add`, `commit` |
| **Week 3** | 버전 관리 기초 | `log`, `status`, 버전 이동(Checkout) |
| **Week 4** | 변경 사항 관리 | `diff`, `.gitignore`, `revert`/`reset` |
| **Week 6** | 원격 저장소 활용 | GitHub 연결, SSH/PAT, `push`/`pull` |
| **Week 7** | 브랜치와 병합 | `branch`, `merge`, 충돌(Conflict) 해결 |
| **Week 8** | 오픈소스 라이선스 | MIT, GPL, Apache, 저작권 이해 |
| **Week 10** | 작업 제어 및 비교 | `stash`, `patch`, 임시 저장 활용 |
| **Week 11** | 커밋 이력 관리 | `rebase`, `cherry-pick`, 히스토리 정리 |
| **Week 12** | 협업 프로세스 | Fork, Pull Request(PR), Issue 관리 |
| **Week 13** | 문서화 및 기여 | README 작성법, Wiki, 기여 가이드라인 |

### 🚀 주요 실습 및 역량 (Highlights)
* **Efficient Version Control:** 복잡한 프로젝트의 히스토리를 체계적으로 관리하고 이전 버전으로 안전하게 복구하는 능력 배양.
* **Collaboration Workflow:** GitHub의 PR 및 Fork 시스템을 활용한 현대적인 오픈소스 협업 워크플로우 숙달.
* **License Compliance:** 프로젝트 성격에 맞는 오픈소스 라이선스를 선택하고 타인의 저작권을 존중하는 윤리적 개발 지식 습득.
* **Branching Strategy:** 기능별 브랜치 운용 및 병합 시 발생하는 충돌을 논리적으로 해결하는 실무 기술 연습.

* `README.md`: 저장소 전체 소개 및 학습 로드맵 요약.
